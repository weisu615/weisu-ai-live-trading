# 国内访问修复方案

## 结论

当前 `weisu.pw` 指向 Render。页面代码已经去掉海外 CDN，图表库也已经本地化，但 Render 的线路在国内移动网络下仍可能无法直连。手机不挂 VPN 进不去，主要不是前端代码问题，而是服务器线路和跨境访问稳定性问题。

## 推荐方案

### 方案 A：阿里云香港 ECS

- 不需要大陆备案即可使用 `weisu.pw`。
- 国内手机访问通常比 Render 稳定。
- 仍然可以运行 Node 服务、Supabase 状态存储、Binance 永续行情拉取。

部署步骤：

1. 购买阿里云香港 ECS，最低 1 核 1G 即可先跑。
2. 安装 Docker。
3. 拉取 GitHub 仓库 `weisu615/weisu-ai-live-trading`。
4. 设置环境变量：
   - `PORT=8787`
   - `NODE_ENV=production`
   - `SYMBOL=BTCUSDT`
   - `STARTING_RMB=1000`
   - `RMB_PER_USDT=7.2`
   - `MAX_STAKE_USDT=20`
   - `MAX_TRADE_HISTORY=100`
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
5. Docker 构建并运行：

```bash
docker build -t weisu-ai-live-trading .
docker run -d --name weisu-ai-live-trading --restart always -p 8787:8787 --env-file .env weisu-ai-live-trading
```

6. 阿里云 DNS：
   - `@` A 记录指向 ECS 公网 IP
   - `www` CNAME 指向 `weisu.pw` 或 A 记录同样指向 ECS 公网 IP
7. 用 Nginx/Caddy 配置 HTTPS。

### 方案 B：阿里云大陆 ECS

- 国内访问最稳。
- 需要先做 ICP 备案，否则域名不能直接解析到大陆服务器提供网站访问。
- 备案完成后，DNS 同样用 A 记录指向大陆 ECS 公网 IP。

## 仍然不能解决的点

如果 Binance 永续行情接口在服务器地区被限制，需要继续使用非限制区域服务器，或者后续接入合规行情代理。代码不会生成假 K 线，行情断开时只保留最后一批真实 K 线。
